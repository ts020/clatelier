# ユーザーストーリー

## Epic: スキル管理

### US-001: スキルの作成
**As a** 開発者
**I want to** GUIでスキルを作成する
**So that** コードを書かずにClaude Codeの能力を拡張できる

**受け入れ条件:**
- [ ] スキル名と説明を入力できる
- [ ] ノードをドラッグ&ドロップで追加できる
- [ ] ノード間をエッジで接続できる
- [ ] 保存するとSKILL.mdが生成される

### US-002: スキルの編集
**As a** 開発者
**I want to** 既存スキルを編集する
**So that** スキルを改善・更新できる

**受け入れ条件:**
- [ ] 既存スキルを読み込める
- [ ] ノードの追加・削除・編集ができる
- [ ] エッジの追加・削除ができる
- [ ] 変更を保存できる

### US-003: スキルのプレビュー
**As a** 開発者
**I want to** 編集中のスキルをプレビューする
**So that** 最終的な出力を確認できる

**受け入れ条件:**
- [ ] リアルタイムでSKILL.md形式をプレビュー
- [ ] バリデーションエラーを表示

## Epic: エージェント管理

### US-010: エージェントの作成
**As a** 開発者
**I want to** 専門エージェントを定義する
**So that** 特定タスクに特化したAIを使える

**受け入れ条件:**
- [ ] エージェント名と説明を入力
- [ ] 許可ツールを選択
- [ ] システムプロンプトを記述
- [ ] 保存するとagent.mdが生成される

## Epic: 開発ワークフロー

### US-020: 機能開発フロー
**As a** 開発者
**I want to** `/feature-dev`で開発を開始する
**So that** 標準化されたプロセスで開発できる

**受け入れ条件:**
- [ ] 設計フェーズが実行される
- [ ] 実装フェーズが実行される
- [ ] レビューがパスするまで繰り返される
- [ ] 完了時にドキュメントが更新される

### US-021: レビューループ
**As a** 開発者
**I want to** レビュー指摘がなくなるまで修正を繰り返す
**So that** 品質基準を満たしたコードを作成できる

**受け入れ条件:**
- [ ] 指摘事項がリスト化される
- [ ] 修正後に再レビューが実行される
- [ ] 全指摘解決で完了となる
